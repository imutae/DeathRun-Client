# DeathRun-Client

> Unity 기반 멀티플레이 클라이언트 프로토타입

## 프로젝트 소개

`DeathRun-Client`는 `DeathRun-Server`와 TCP 소켓으로 통신하는 Unity 클라이언트 프로젝트입니다.

처음에는 2D 플랫포머 데스런 게임을 목표로 기획했지만, 현재는 포트폴리오 목적에 맞춰 범위를 조정하여 서버와 연동되는 멀티플레이 클라이언트 구조 구현에 집중하고 있습니다.

현재 중점적으로 구현한 영역은 다음과 같습니다.

- 서버 접속 및 세션 식별자 수신
- 로비 UI와 채팅 흐름
- 방 목록 요청, 방 생성, 방 참가 요청 흐름
- 게임 씬 이동을 위한 기반 구조
- 플레이어 이동, 점프, 애니메이션 기본 컨트롤러
- TCP 기반 패킷 빌드, 파싱, 직렬화 구조

이 프로젝트는 완성형 게임 클라이언트라기보다, 서버와 맞물리는 클라이언트 구조를 어떻게 설계하고 있는지 보여주기 위한 포트폴리오 프로젝트입니다.

---

## 개발 목적

- C++ 서버와 통신하는 Unity 클라이언트 직접 구현
- TCP stream 기반 패킷 파싱 구조 설계
- 패킷 직렬화 / 역직렬화 구조 설계
- 로비 씬과 게임 씬의 상태 흐름 정리
- 채팅 UI, 풀링, 매니저 구조 등 재사용 가능한 클라이언트 기반 구성
- 서버 개발자 관점에서 클라이언트-서버 프로토콜 함께 검증

## 현재 구성된 시스템

### 1. 씬 구성

현재 프로젝트에는 아래 두 개의 핵심 씬이 포함되어 있습니다.

- `LobbyScene`
  - 서버 접속
  - 로비 UI
  - 채팅 UI
  - 방 목록 / 방 생성 / 방 참가 요청 흐름
- `GameScene`
  - 실제 플레이 씬 확장을 위한 기반 씬
  - 로컬 플레이어 이동 / 점프 / 애니메이션 테스트

### 2. 네트워크 매니저

`NetworkManager`는 서버 연결과 패킷 처리를 관리합니다.

주요 역할은 다음과 같습니다.

- 서버 접속 / 연결 종료
- 수신 패킷 큐 관리
- Unity 메인 스레드에서 패킷 처리
- 채팅, 접속 승인, 방 목록, 방 참가, 이동 패킷 분기 처리
- 서버에서 받은 `sessionId`를 로컬 플레이어 식별 정보로 저장
- UI 계층에 필요한 이벤트 전달

### 3. TCP 클라이언트 코어

`TcpClientCore`는 실제 소켓 연결과 비동기 수신 루프를 담당합니다.

- 서버 연결 / 종료
- async read 기반 수신 루프
- 수신 바이트를 `PacketParser`에 누적
- 완성된 패킷을 상위 계층에 이벤트로 전달
- disconnect / error 이벤트 분리

### 4. 패킷 계층

서버와 동일한 방향으로 `[size][id][body]` 구조의 패킷을 처리하기 위해 패킷 계층을 분리했습니다.

- `PacketBuilder`
  - 패킷 헤더와 body를 조합하여 전송 가능한 byte 배열 생성
- `PacketParser`
  - TCP stream에서 완성된 패킷 단위로 분리
- `PacketSerializer`
  - 기본 타입과 고정 길이 문자열 직렬화 / 역직렬화
- `Protocol/*`
  - `ChatPacket`, `SRoomListPacket`, `SJoinPacket`, `EAcceptPacket` 등 개별 패킷 정의

### 5. 로비 UI

로비에서는 서버와 연결된 UI 흐름을 구성하고 있습니다.

- 방 목록 요청
- 방 생성 요청
- 방 참가 요청
- 서버에서 받은 방 목록을 버튼 UI에 반영
- 채팅 메시지 송신 / 수신 / 표시

### 6. 채팅 UI

`ChatUI`와 `ChatElement`를 통해 로비 채팅을 구성했습니다.

- 입력 필드에서 메시지 전송
- 서버 수신 채팅 패킷을 UI에 반영
- 채팅 엘리먼트 프리팹 동적 생성
- 풀링 구조와 연결하여 UI 오브젝트 재사용 가능하도록 설계

### 7. 오브젝트 풀링

`PoolManager`와 `IPoolable` 기반으로 UI/게임 오브젝트 재사용 구조를 구성했습니다.

- Addressables 기반 프리팹 로드
- 프리로드 지원
- 활성 / 비활성 오브젝트 관리
- `OnSpawn`, `OnDespawn` 콜백 기반 초기화 흐름

### 8. 플레이어 기본 컨트롤러

`GamePlay/Player`에는 아래 기본 기능이 포함되어 있습니다.

- 좌우 이동: `PlayerMovement`
- 점프: `PlayerJump`
- 애니메이션 상태 제어: `PlayerAnimation`

아직 완성된 멀티플레이 동기화 구조는 아니지만, 이후 `R_MOVE` / `S_MOVE` 패킷과 연결하기 위한 기반으로 작성했습니다.

## 구현 현황

| 기능 | 상태 | 설명 |
| --- | --- | --- |
| 서버 접속 | 구현 | `NetworkManager`에서 로컬 서버 접속 |
| 접속 승인 처리 | 구현 | `E_ACCEPT` 수신 후 `sessionId` 저장 |
| 채팅 송신 / 수신 | 구현 | 입력, 전송, 수신, UI 표시 흐름 구성 |
| 패킷 빌드 / 파싱 | 구현 | `size + id + body` 기반 패킷 처리 |
| 로비 씬 / 게임 씬 분리 | 구현 | 기본 씬 구조 분리 |
| 방 목록 요청 / 표시 | 진행 중 | `S_ROOM_LIST` 기반 UI 표시 흐름 구성 |
| 방 생성 / 참가 요청 | 진행 중 | 서버 프로토콜과 연결되는 요청 흐름 구성 |
| 게임 씬 전환 | 진행 중 | 방 참가 성공 시 게임 씬으로 이동하는 기반 구성 |
| 로컬 플레이어 이동 / 점프 / 애니메이션 | 구현 | 기본 조작 테스트 가능 |
| 위치 동기화 | 진행 중 | `R_MOVE` / `S_MOVE` 연동 기반 구성 |
| 원격 플레이어 표시 | 예정 | `sessionId` 기반 오브젝트 관리 예정 |
| leave / disconnect UI 처리 | 예정 | 서버 응답과 UI 상태 연동 필요 |

## 실행 흐름

1. Unity에서 프로젝트 실행
2. `LobbyScene` 진입
3. `NetworkManager`가 서버에 접속
4. 서버로부터 `E_ACCEPT` 수신
5. 로컬 플레이어의 `sessionId` 저장
6. 로비 UI에서 채팅 / 방 목록 요청 / 방 생성 / 방 참가 요청 수행
7. 서버 수신 패킷을 큐에 저장
8. Unity 메인 스레드에서 패킷 처리
9. 방 참가 성공 시 `GameScene`으로 이동
